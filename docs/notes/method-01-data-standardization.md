# M01 数据标准化与异常场

## 对应代码

- `src/tropical_wave_tools/io.py`
- `src/tropical_wave_tools/preprocess.py`
- `src/tropical_wave_tools/preprocessing.py`

## 解决的问题

热带波诊断首先需要把不同来源的 NetCDF 数据整理成统一的时空场。当前项目默认把输入整理为：

```text
X(time, lat, lon)
```

其中 `time` 是时间维，`lat` 是纬度，`lon` 是经度。常见别名如 `valid_time`、`latitude`、`longitude` 会被自动标准化。

## 数学原理

最基础的异常场定义为：

```text
X'(t, y, x) = X(t, y, x) - C(g(t), y, x)
```

其中：

- `X` 是原始变量，例如 OLR、降水、U850、V850。
- `C` 是气候态。
- `g(t)` 可以是月份、季节或一年中的第几天。

如果使用日气候态并保留低阶谐波，可以写成：

```text
C(d) ~= a0 + sum_{n=1..N} [an cos(2*pi*n*d/365) + bn sin(2*pi*n*d/365)]
```

当前滤波流程默认使用日气候态的低阶谐波来表示平滑年循环。

## 物理意义

去掉平均季节循环后，剩下的异常场更接近天气尺度到季节内尺度的扰动。对 OLR 来说，负异常通常代表深对流增强；对降水来说，正异常通常代表降水增强；对风场来说，正负号代表方向或环流结构。

## 实现要点

`load_dataarray()` 会完成：

1. 读取 NetCDF。
2. 选择变量名。
3. 标准化坐标名。
4. 经度转换到 `0..360`。
5. 纬度排序。
6. 选择时间、纬度、经度范围。
7. 确认最终维度为 `(time, lat, lon)`。

如果数据有单层额外维度，例如 `level=850`，会自动压缩；如果有多个高度层，则需要先在外部或代码中选择层。

## 注意事项

- OLR 的事件符号和降水相反。OLR 深对流常用负异常，降水深对流常用正异常。
- WK 谱和滤波默认假设数据采样间隔与 `samples_per_day` 或 `--spd` 一致。
- 若经纬度不是规则网格，风场散度和涡度诊断需要额外谨慎。
- 缺测值会影响 FFT、EOF 和回归结果，进入谱分析前应尽量处理。
